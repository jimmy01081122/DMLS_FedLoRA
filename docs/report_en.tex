\begin{abstract}
This study investigates three LoRA-based federated fine-tuning variants---Standard LoRA, FFA-LoRA, and RoLoRA---in a hardware-constrained setup using a single NVIDIA RTX 3050 GPU. We evaluate these methods on the AG News task using the Qwen2.5-1.5B model. Our findings indicate that Standard LoRA converges rapidly with moderate data sizes (300 samples/round), while FFA-LoRA and RoLoRA offer significant communication savings of up to 62\% despite requiring more rounds for full convergence.
\end{abstract}

\section{Introduction}
Federated Learning (FL) combined with Parameter-Efficient Fine-Tuning (PEFT) enables LLM adaptation on decentralized data. This research compares different LoRA strategies to optimize communication efficiency and aggregation stability on consumer-grade hardware.

\section{Methodology}
\subsection{Model and Task}
We employ Qwen2.5-1.5B with 4-bit NF4 quantization. The task is a 4-class sequence classification on AG News. 

\subsection{Federated Learning Methods}
\begin{itemize}
    \item \textbf{Standard LoRA}: Updates and transmits both $A$ and $B$ matrices.
    \item \textbf{FFA-LoRA}: Matrix $A$ is frozen; only $B$ is trained.
    \item \textbf{RoLoRA}: Alternates the training of $A$ and $B$ across rounds.
\end{itemize}

\subsection{Data Scaling Analysis}
We compared "Quick Mode" (10 samples/round) and "Balanced Mode" (300 samples/round):
\begin{itemize}
    \item \textbf{Low Data (10 samples)}: All methods failed to converge, showing random accuracy (~25\%).
    \item \textbf{Moderate Data (300 samples)}: Standard LoRA reached 87.53\% accuracy within only 5 global rounds.
\end{itemize}

\section{Conclusion}
RoLoRA and FFA-LoRA provide significant communication efficiency. However, Standard LoRA remains the most robust choice under low-round constraints. The RTX 3050 (6GB) is proved sufficient for 1.5B model FL.
